Billing FAQs for Vendors

Below are some commonly asked questions by Vendor customers.


Q: How do I see pricing for the various add-on options?


Pricing calculations are all built into the Billing page :)
In the Billing page, if you hit the "Buy" button next to any add-on option then you'll be able to see the monthly and annual fees payable.
The Buy action is only applied when you hit the Save button - and there is a Confirm step thereafter - so there is no risk of immediate purchase when you hit the Buy button.



Q: What is the purpose of the user purchase option in the Billing area?  Why does it only give "Annually" as an option?


The option to purchase annual users as a Vendor is specifically so that you can take advantage of prepaid annual discounts.On old grandfathered plans, paying for users annually was never an option - with our new model, now it is :)

NOTE: If you prepay for annual users on a client, they can still exceed that annual license count - any additional activated users above the prepaid number will simply be billed at monthly rates in arrears as normal.

This is handy if you have a known, fixed number of annual users but need to allow for additional users as needed.

See the question below for details of how to enforce user/device limits.



Q: How do I limit the number of users or devices that a client can add to their account?


If you want to specifically limit the maximum number of users, you can do so via:

  • Our Company API, or
  • Via the Organisation Setup page - when logged in as a Vendor Admin, you'll see various user limit options on the right hand side.



Q: How do I activate a trial of an add-on for a client?


You do this via the Billing page; the process is the same for every add-on.  

  1. Go to the Billing page, and select the client from the "Show Prepaid Billing For" dropdown
  2. After the page refreshes, hit the Try button next to the Enterprise Toolkit add-on
  3. You will see instructions on what to do next in the activation page


Thereafter for Enterprise Toolkit:

  1. Log into the client account by going to the Clients page and switching context into the client
  2. Grant the Enterprise Admin user level to the desired target users.
    Those users will then see the new Environments and Repository options in the Menu.
  3. Notify your client as desired of the add-on availability.


Thereafter for Sync:

You'll need to download the Sync install and provide this, along with the Sync license key, to your client for them to install.

You may also want to adjust the colour scheme of the Sync install, which is easily done by changing the css file within the install zip.


Thereafter for App as a Service:

Gather the required elements as outlined in the activation page, and provide those to us for building of the trial apps.



Q: Can I upload my own customisations to the Sync installation file?


Currently there is no option to inject your own branded artefacts into the Sync zip file.
Sync is a standard download though, so you can download it once and distribute to clients as needed.
You'll only need to update the Sync zip when we release new Sync versions, which will be announced via blog and email.


Note: You'll still need to activate a trial or purchase Sync per client in order to get a license key for each client.



Q: If I activate a trial of an add-on, e.g. Sync, from the Billing area, how does my client get notified?


We don't send anything to your client when you activate a trial of Sync or any other add-on for a client account.

The expectation is that you, as the Vendor, will download the Sync package and provide it, along with the Sync key, to your client as desired.

You may also want to adjust the colour scheme of the Sync install, which is easily done by changing the css file within the install zip.


For other add-ons the experience is similar - i.e. nothing is sent to the client by us, we expect you as the Vendor to be the intermediary.  

In the case of the Enterprise Toolkit, activation will immediately expose the Environments and Repository options in the platform's Menu.  
This will be visible to Enterprise Admin users of the target client - so they may notice that :)
