AaaS Publishing Requirements for Windows

The only requirement is to brand the app according to the App Checklist via the Branding Center when logged into the web portal, during a trial, or after purchasing.



Our Windows apps are provided as a hosted file download as opposed to a Windows Store distribution. This is because we support Windows 7, which does not have access to the Windows Store. 

Despite this, our Windows apps include seamless auto-update functionality and are simple one-click installs. We will explore a Windows 8/10 Store option in the future based on customer demand.



Publishing a Windows branded app is a 2 business day turnaround on Your Companys end, we provide you with an executable install file directly to reduce access issues if clients do not have a Microsoft account to access Windows Store.
